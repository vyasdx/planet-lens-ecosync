## 🎯 Challenge 3 — Carbon Footprint Awareness Platform

> *"Design a solution that helps individuals understand, track, and reduce their carbon
> footprint through simple actions and personalized insights."*

Rather than a plain logging dashboard, Planet Lens reframes carbon awareness as a
**mission** — a Captain Planet–inspired command center where you restore a grayed-out,
struggling Earth back to full color by making real low-carbon choices. This turns an
abstract metric into an emotional, habit-forming experience.

### How it maps to the problem statement

| Requirement | How Planet Lens delivers it |
| :--- | :--- |
| **Understand** | A live 3D Earth, real-time **global atmospheric CO₂** (Mauna Loa data), an emissions breakdown doughnut, and a Mother Earth Bot that explains your impact in plain language. |
| **Track** | A personal footprint that updates as you act, daily/weekly/monthly EcoPoints, a monthly 25 kg CO₂e mission, historical trend chart, and a 5-ring progress system. |
| **Reduce** | **AI Decision Lenses** intercept everyday choices (food, commute, shopping, waste, energy) at the moment of decision and nudge to lower-carbon swaps; a 25-action library turns intent into verified habits. |
| **Personalized insights** | A **"My Lifestyle"** calculator builds your real footprint from your vehicle, diet, energy, and travel; EcoBot generates tailored recommendations from your data. |

### Chosen vertical & persona

**Vertical:** Carbon Footprint Awareness Platform (Challenge 3).

**Persona — "the well-intentioned but disengaged individual":** someone who *wants* to live
more sustainably but finds carbon tools abstract, guilt-inducing, or easy to abandon. They
don't need another spreadsheet — they need motivation, clarity at the moment of choice, and a
visible sense of progress. Planet Lens is designed around this persona: it meets them at
everyday decisions (food, commute, shopping, waste, energy), rewards action with visible
planetary restoration, and keeps the data personal and private.

**Decision logic (context-aware):** the assistant reasons over the user's region, active
sensors, current footprint, and which rings remain locked to decide what to surface — which
swap to suggest, which verification method to request (camera/GPS/motion vs. manual), and which
element a completed action restores. See [HOW_IT_WORKS.md](HOW_IT_WORKS.md) for the full pipeline.

## 🌟 Key Features

- **Command-Center UI** — persistent navigation (Dashboard / Action Library / Community / Terminal / Logs) with an Eco-Futurism glassmorphic design system.
- **Cinematic 3D Earth** — a colorful living Earth on the welcome gate flies into the dashboard and **desaturates to grayscale**, then returns to full color as you restore the 5 Planeteer rings (Earth, Fire, Wind, Water, Heart).
- **AI Decision Lenses** — moment-of-decision interception for Food, Commute, Shopping, Waste, and Energy, each running a multi-agent pipeline (Context → Carbon → Swap → Nudge → Verify → Gaia).
- **25-Action Element Library** — one-time, habit, and long-term actions across the five elements, each with CO₂ saved, points, and a verification method.
- **Real device-sensor verification** — camera (`getUserMedia`), live GPS (`geolocation`), and motion/step counting (`DeviceMotion`) to prove actions.
- **Live global CO₂** — real-time atmospheric ppm from the global-warming.org API, with graceful offline fallback.
- **"My Lifestyle" footprint calculator** — personalizes the carbon math to the actual user.
- **Mother Earth Bot Terminal** — conversational guidance + Planeteer trivia.
- **"Go Planet!" voice** — synthesized Captain Planet line (Web Speech API) when all 5 rings activate.
- **Community feed & System Logs** — planetary activity stream and telemetry.

## 🧠 AI & Prompt Engineering (the Prompt War core)

Personalized reduction insights are generated by **Google Gemini 1.5 Flash**, integrated for
**production reliability** — not a raw client-side call:

- **Server-side proxy** (`/api/insights` in `server.js`) — the API key lives only on the server
  (read from `GEMINI_API_KEY` / Secret Manager) and is **never exposed to the browser**.
- **Structured output** — Gemini is constrained by a strict `responseSchema` + `responseMimeType:
  application/json`, so every response is a validated array of `{category, suggestion,
  estimated_saving_kg}` (see `gemini.js`).
- **Low temperature (0.4)** — factual, consistent, non-exaggerated recommendations.
- **System instruction** that forces specific, quantified, source-prioritized advice over generic tips.
- **Deterministic fallback engine** — if the model is unavailable, times out (8s), or returns
  malformed output, a rule-based engine instantly produces valid suggestions, so the feature
  **never hard-fails**.
- **Rate limiting** (per-IP) and **payload caps** on the endpoint.

The prompt builder, fallback engine, and response sanitizer are pure functions with unit tests
(`tests/gemini.test.js`).

## 🏗️ Tech & Architecture

Built with **vanilla HTML / CSS / JavaScript and zero runtime dependencies** — no framework,
no build step. This keeps the bundle tiny, the code transparent, and the app instantly portable.

| File | Responsibility |
| :--- | :--- |
| `index.html` | App shell (sidebar + top bar + bottom nav), all six screens, modals. |
| `styles.css` | Eco-Futurism design tokens, glassmorphism, responsive + mobile/PWA layout. |
| `app.js` | State machine, carbon engine, decision pipelines, sensor verification, Three.js Earth, Web Audio, and a clearly-separated enhancement layer for the command-center features. |
| `core.js` | Pure, DOM-free carbon logic (regional baselines, footprint, projections) — unit-tested. |
| `server.js` | Static server with directory-traversal protection + the `/api/insights` Gemini proxy (rate-limited). |
| `gemini.js` | Server-side Gemini 1.5 Flash client: prompt builder, response schema, sanitizer, fallback engine. |
| `manifest.json` | PWA manifest for installability (standalone, themed). |

- **State & privacy:** all user data lives in `localStorage` on the device — nothing is uploaded to a server.
- **Security:** user-supplied text is escaped (`escapeHTML`) before rendering; no `eval`/dynamic code execution; the static server rejects path-traversal.
- **Accessibility & reach:** semantic landmarks, `aria-label`s on icon controls, image `alt` text, keyboard-reachable controls, and a responsive layout with an Android-style bottom nav and safe-area handling on mobile.

## 🛠️ Run Locally

Requires Node.js 18+ (no packages to install — zero dependencies).

```bash
# Optional: enable live Gemini insights (falls back to the rule engine if unset)
export GEMINI_API_KEY=your_key_from_aistudio.google.com/apikey

npm start
# serves on http://localhost:8080
```

Without a key the app runs fully — the AI insights panel uses the deterministic fallback engine.

## ✅ Testing

Core carbon logic lives in `core.js` — a pure, DOM-free, dependency-free module that the
browser app and the test suite both consume. Tests use Node's built-in runner (no frameworks):

```bash
npm test
```

`tests/core.test.js` covers regional baselines, the personal footprint calculator
(`computeFootprint`), projected emissions with ring reductions and offsets, planet-health,
offset points, regional comparison, tree-offset math, rank thresholds, the ISO week key, and
`escapeHTML` (XSS-safety). `tests/gemini.test.js` covers the prompt builder, the deterministic
fallback engine, response sanitization, and the no-key fallback path — **25 passing tests**
total. Separating pure logic into `core.js` / `gemini.js` keeps the carbon and AI engines
independently testable and the UI layer thin.

## 📌 Assumptions

- **Single-user, client-side prototype.** All state is per-device in `localStorage`; there is no
  backend, login, or multi-user sync (privacy-first by design).
- **Emission factors are representative averages**, not audited per-region figures — chosen to be
  defensible and to make relative comparisons meaningful (see `core.js`).
- **Action verification is proof-of-intent**, not third-party-audited proof — device sensors
  (camera/GPS/motion) demonstrate the action was plausibly taken.
- **Community feed, System Logs, and Net-Zero simulation are illustrative demo data** to show the
  intended at-scale experience; global CO₂ and device sensor data are real (see Data Transparency).
- Modern browser assumed (ES2017+, WebGL, Web Audio, Web Speech); features degrade gracefully when
  a capability or network is unavailable.
